# AGENTS.md — nv-designer

## Scope
Visual form designer tool that reads a form description and emits pure C source
files. The output is human-readable, version-control-friendly C code that calls
the `nv-ui` public API. There is no intermediate `.nvui` format — the designer
writes C directly.

`nv-ui` is the only dependency. `nv-designer` does NOT link the full platform
stack at design time; it only needs `nv-ui` headers for type information.

## Owns
- `src/main.c`          — tool entry point (CLI / embedded UI)
- `src/nv_dsg_parser.c` — reads the designer's internal project format
- `src/nv_dsg_emit.c`   — emits `myform_ui.c` (designer-owned block)
- `src/nv_dsg_merge.c`  — round-trip: preserves user code in `myform.c`

## Two-File Output Pattern
For every form `myform` the designer produces exactly two files:

```
myform_ui.c   ← designer-owned
myform.c      ← user-owned (created once, never overwritten)
```

### `myform_ui.c` (designer-owned)
```c
/* NV_DESIGNER_BEGIN — generated by nv-designer vX.Y. DO NOT EDIT. */

#include "nv_ui.h"

void myform_ui_create(nv_form_t* form) {
    nv_button_t* btn_ok = nv_button_new(form, &(nv_button_cfg){
        .caption = "OK",
        .x = 100, .y = 200, .w = 80, .h = 30,
    });
    nv_label_t* lbl_title = nv_label_new(form, &(nv_label_cfg){
        .text = "Hello",
        .x = 10, .y = 10,
    });
    (void)btn_ok; (void)lbl_title;
}

/* NV_DESIGNER_END */
```

### `myform.c` (user-owned)
```c
#include "myform_ui.c"   /* pulls in the generated creation sequence */

/* User event handlers live below — nv-designer never touches this file. */

static void on_btn_ok_click(nv_button_t* btn, void* ctx) {
    (void)btn; (void)ctx;
    /* TODO: implement */
}

nv_form_t* myform_create(nv_app_t* app) {
    nv_form_t* form = nv_form_new(app, &(nv_form_cfg){
        .title = "My Form", .w = 400, .h = 300,
    });
    myform_ui_create(form);
    nv_button_on_click(nv_form_find(form, "btn_ok"), on_btn_ok_click, NULL);
    return form;
}
```

## NV_DESIGNER_BEGIN / NV_DESIGNER_END Contract
1. The designer ONLY rewrites the region between `NV_DESIGNER_BEGIN` and
   `NV_DESIGNER_END` in `myform_ui.c`.
2. Everything outside that region in `myform_ui.c` (file-level includes,
   the closing `/* NV_DESIGNER_END */`) is also regenerated in full on each
   save — treat the entire `myform_ui.c` as designer-owned.
3. `myform.c` is NEVER modified after initial creation.
4. The merge engine (`nv_dsg_merge.c`) verifies that the user has not placed
   code inside `myform_ui.c` before overwriting it. If user edits are detected
   it aborts with a clear error rather than silently discarding them.
5. Component names used in `myform_ui.c` (e.g. `btn_ok`) are stable identifiers
   that the user references in `myform.c`. Renaming a component in the designer
   is a refactor — the tool must update `myform.c` references or warn.

## Allowed dependencies
- `nv-ui` (headers + types only at design time)
- C11 stdlib (file I/O, string processing)

## Forbidden
- Do NOT link `nv-platform-*` or `nv-runtime` into the designer executable
- Do NOT produce `.nvui` or any binary intermediate format
- Do NOT overwrite `myform.c` after its initial creation
- Do NOT silently discard user edits inside `myform_ui.c`
- Do NOT change the LICENSE file

## Coding rules
- C11, `-Wall -Wextra` clean
- Max 300 lines per `.c` file — split if exceeded
- All emitted C code must compile with `-Wall -Wextra -Werror` against nv-ui
- Round-trip test: parse → emit → parse must produce identical output
