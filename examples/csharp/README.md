# C# examples — nativeview

> **Status:** The **minimal** sample here is experimental (see [docs/CSharp.md](../../docs/CSharp.md#known-limitations)). The full Vue todo port at **`examples/todo_app/csharp_todo/`** is **not working end-to-end yet**.

**macOS / Linux:** `build_static.sh` links the static archive chain like the Nim and C examples. **Windows:** use **`build_static.ps1`** (MSVC + WebView2 static loader).

## Files

| File | Purpose |
|------|---------|
| `Minimal/Program.cs` | Tiny app: version string, `nv_on_ready` → `nv_load_html`, `nv_eval_js_batch`, `nv_app_destroy`. |
| `build_static.sh` | CMake static build + `dotnet publish` — Linux and macOS. |
| `build_static.ps1` | CMake static build + `dotnet publish` — Windows MSVC. |
| `NativeView.Static.props` | Generated by the build scripts (not checked in). |

Full linking rules: **`docs/CSharp.md`**. Binding: **`bindings/csharp/NativeView.cs`**.

## Quick build (Linux)

From **`examples/csharp`**:

```sh
./build_static.sh
```

Override the CMake build tree:

```sh
NV_CMAKE_BUILD_DIR=/path/to/build ./build_static.sh
```

## Quick build (Windows)

```powershell
.\build_static.ps1
```

If `dotnet publish` reports **No .NET SDKs were found** while the SDK is installed, Windows
often has a **runtime-only** `dotnet` under `Program Files (x86)` ahead of the real SDK on
`PATH`. Fix: put `C:\Program Files\dotnet` first, uninstall the x86 runtime shim, or run:

```powershell
$env:DOTNET_EXE = 'C:\Program Files\dotnet\dotnet.exe'
.\build_static.ps1
```

## Shared build (all platforms)

```sh
cmake -S ../.. -B ../../build-csharp-shared -DNV_BUILD_SHARED=ON -DNV_BUILD_TESTS=OFF -DNV_ENFORCE_FILE_LIMITS=OFF
cmake --build ../../build-csharp-shared --target nativeview_shared -j8
export LD_LIBRARY_PATH="../../build-csharp-shared:$LD_LIBRARY_PATH"   # Linux
dotnet publish Minimal/Minimal.csproj -c Release -p:DefineConstants=NATIVEVIEW_SHARED
```

On Windows, ensure **`nativeview.dll`** is on **`PATH`** or next to **`minimal.exe`**.
